Data Base Utility User Guide

This user guide provides a detailed explanation of how to use the Data Base Utility application. The application is designed to manage and manipulate data related to call signs, QTH (location), names, statuses, and other related fields. It includes features for loading, saving, editing, and analyzing data, as well as managing sweepers and attendance records.

Data Base Utility System Requirements

1. Introduction

The Data Base Utility is a Tkinter-based application that allows users to manage and analyze data for radio communication networks. It supports features such as:

  • Loading and saving data in CSV format.
  • Editing and updating data entries.
  • Managing sweepers and attendance records.
  • Generating and archiving monthly rosters.
  • Support for dark mode and customizable font sizes.

2. Getting Started

To start using the application, ensure that the following prerequisites are met:

  • Python 3.x is installed on your system.
  • Required Python libraries are installed (e.g., tkinter, csv, datetime).
  • The application is launched by running the script: python DataBaseUtility.py.

Once the application is running, the main interface will appear.

3. Main Interface

The main interface of the application consists of the following components:

  • Menu Bar: Contains options for file management, data management, and settings.
  • Toolbar: Provides quick access to frequently used actions such as searching, refreshing, and resetting the table.
  • Tree View: Displays the data in a tabular format with columns for Call Sign, QTH, Name, Status, and other fields.
  • Stats View: Shows statistics such as attendance counts, visitor counts, and total time elapsed.
  • Sweepers View: Displays a list of sweepers and their details.

4. Toolbar

The toolbar is located at the top of the application and provides quick access to the following actions:

  • Search:
    • Click this button to search for specific data in the table. A dialog box will appear where you can enter your search term.
    • If the search term is found, the corresponding row will be highlighted. If no results are found, you can choose to add a new row with the search term.
  • Refresh:
    • Click this button to refresh the table and update the statistics. This action recalculates totals for attendance, sweepers, and other metrics.
    • The table will also be sorted alphabetically by the "Call Sign" column.
  • Reset:
    • Click this button to reset the table to its default state. This action clears all attendance records, resets statuses to "Absent," and removes sweepers.
    • The table will be sorted alphabetically by the "Call Sign" column, and focus will be set to the first row.

The toolbar buttons are designed to streamline common tasks and improve the user experience.

5. Menu Options

5.1 File Menu

  • Open: Load data from a CSV file.
  • Save: Save the current data to a CSV file.
  • Data Management:
    • Create Monthly Roster: Generate a new monthly roster.
    • Archive Monthly Roster: Archive the current monthly roster.
  • Exit: Close the application after confirming unsaved changes.

5.2 Options Menu

  • Font Size: Change the font size of the interface (Small, Medium, Large).
  • Preamble: Load preamble text for different types of nets (e.g., Roll Call Net, Open Net).
  • Net Type: Select the type of net (Roll Call Net or Open Net).
  • Toggle Dark Mode: Enable or disable dark mode for the interface.

6. Working with Data

6.1 Loading Data

To load data into the application:

  1. Go to the File menu and select Open.
  2. Choose a CSV file from your system.
  3. The data will be displayed in the Tree View.

6.2 Saving Data

To save the current data:

  1. Go to the File menu and select Save.
  2. Choose a location and filename for the CSV file.
  3. The data will be saved to the specified file.

6.3 Editing Data

To edit data in the Tree View:

  • Double-click on a cell to edit its value.
  • Use the context menu (right-click) to insert, delete, copy, or paste rows.
  • Click on certain columns (e.g., Status, Relayed, Traffic) to update their values interactively.

6.4 How To: Create a New Roll Call Net

To create a new Roll Call Net, follow these steps:

  1. Load CSV File:
    • Go to the File menu and select Open.
    • Load a properly formatted CSV file that contains the roster data.
  2. Press Reset:
    • Click the Reset button on the toolbar to clear any existing data and reset the table to its default state.
  3. Select Net Type:
    • Go to the Options menu and click Net Type.
    • Select Roll Call Net from the dropdown menu.
  4. Enter Required Inputs:
    • A dialog box will appear asking you to input the necessary information for the net name.
    • Enter the name of the Roll Call Net and click OK.
  5. Refresh:
    • Click the Refresh button on the toolbar to update the table and recalculate statistics.
  6. Save File:
    • Go to the File menu and select Save.
    • Save the file using the desired name and location.

Once these steps are completed, your new Roll Call Net will be ready for use.

6.5 How To: Create a New Open Net

To create a new Open Net, follow these steps:

  1. Load CSV File:
    • Go to the File menu and select Open.
    • Load a properly formatted CSV file that contains the roster data.
  2. Press Reset:
    • Click the Reset button on the toolbar to clear any existing data and reset the table to its default state.
  3. Select Net Type:
    • Go to the Options menu and click Net Type.
    • Select Open Net from the dropdown menu.
  4. Enter Required Inputs:
    • A dialog box will appear asking you to input the necessary information for the net name.
    • Enter the name of the Open Net and click OK.
  5. Refresh:
    • Click the Refresh button on the toolbar to update the table and recalculate statistics.
  6. Save File:
    • Go to the File menu and select Save.
    • Save the file using the desired name and location.

Once these steps are completed, your new Open Net will be ready for use.

7. Managing Sweepers

The application allows you to manage sweepers (individuals responsible for sweeping the net):

  • Click on the Sweep? column to mark a row as a sweeper.
  • The Sweepers View will display the list of sweepers.
  • Use the context menu in the Sweepers View to clear or delete sweepers.

8. Attendance and Stats

The application tracks attendance and provides statistics:

  • The Stats View displays counts for Present, Excused, and Visitor attendees.
  • The total time elapsed for the net is also shown.
  • Attendance records can be archived or reset as needed.

9. Dark Mode

The application supports a dark mode for better visibility in low-light environments:

  • Go to the Options menu and select Toggle Dark Mode.
  • The interface will switch between light and dark themes.

10. Understanding File Headers

The application uses specific headers in CSV files to organize and interpret data. One such header is:

"Time Start,Time End,Total Present,Current Net Name,Total Excused,Net Control"

This header is used to store metadata about the net session, including the start and end times, attendance counts, and the net control operator. Below is an example of how this header is used in a CSV file:

Time Start,Time End,Total Present,Current Net Name,Total Excused,Net Control
2024-12-14 18:00:05,2024-12-14 18:25:53,0,Sample Roll Call Net,0,NL7PA

10.1 Field Descriptions

  • Time Start: The timestamp when the net session began. Format: YYYY-MM-DD HH:MM:SS.
  • Time End: The timestamp when the net session ended. Format: YYYY-MM-DD HH:MM:SS.
    • The total time the network was active is calculated based on the difference between the Time Start and Time End. For example:
      • If Time Start = 2024-12-14 18:00:05 and Time End = 2024-12-14 18:25:53, the total time elapsed is 00:25:48 (25 minutes and 48 seconds).
  • Total Present: The total number of attendees marked as "Present" during the net session.
  • Current Net Name: The name of the net session (e.g., "Sample Roll Call Net").
  • Total Excused: The total number of attendees marked as "Excused" during the net session.
    • There are two reasons a person may be marked as "Excused":
      1. Request to be Excused: If the person checks in and then requests to be excused, their present count will increment by +1.
      2. Vacation or Other Reasons: If the person is excused for vacation or other reasons, they will be marked as "Excused" but will not receive credit.
  • Net Control: The call sign of the net control operator (e.g., "NL7PA").

10.2 Example Data

In the example above:

  • The net session started at 2024-12-14 18:00:05 and ended at 2024-12-14 18:25:53.
  • The total time elapsed is 00:25:48 (25 minutes and 48 seconds).
  • There were 0 attendees marked as "Present" and 0 marked as "Excused".
  • The net was named "Sample Roll Call Net".
  • The net control operator was "NL7PA".

11. Troubleshooting

If you encounter any issues while using the application, refer to the following troubleshooting scenarios:

  • Loading the Wrong Type of File:
    • If you attempt to load a file that is not in the correct CSV format, the application will display an error message: "Failed to open file: [error message]".
    • Ensure that the file you are loading is a valid CSV file with the correct headers and data structure.
  • Editing When No Data is in the Table:
    • If you try to edit a cell or perform an action (e.g., insert a row) when no data is loaded in the table, the application will display a warning message: "No Data: Please load data first.".
    • Load a valid CSV file before attempting to edit or modify the data.
  • Saving When No Data is Loaded:
    • If you attempt to save the data when no data is loaded in the table, the application will display a warning message: "No Data: Please load data first.".
    • Load a valid CSV file before attempting to save the data.

If you encounter any other issues, ensure that the required Python libraries are installed, check the CSV file format for compatibility, and refer to the application logs for error details.
